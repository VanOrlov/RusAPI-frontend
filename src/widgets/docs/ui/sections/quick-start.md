Всего 3 простых шага, и у вас появится полноценный REST API для вашего фронтенд-приложения. Никаких установок на локальный компьютер, баз данных и Docker-контейнеров.

## Шаг 1. Создайте проект
Проект — это изолированное пространство для вашего приложения (например, "Интернет-магазин" или "Таск-трекер").

1. Перейдите на вкладку **Проекты**.
2. Нажмите кнопку **Создать проект** и задайте ему имя.
3. После создания система выдаст вам уникальный идентификатор — `nanoId` (например, `abc123xyz`). Этот ID будет частью вашего базового URL.

## Шаг 2. Создайте эндпоинт (ресурс)
Ресурс — это сущность, с которой вы будете работать (например, `users`, `products`, `tasks`).

1. Откройте ваш проект и нажмите **Создать эндпоинт**.
2. Укажите название пути (только английские буквы, например `users`).
3. **Настройте схему данных:** добавьте нужные поля и выберите для них тип генерации (например, для поля `avatar` выберите тип `image.avatar`).
4. Нажмите **Сохранить**. 

> **💡 Примечание:** Поле `id` (уникальный UUID) создается **автоматически** для каждой записи. Вам не нужно добавлять его в схему вручную! Как только вы создали эндпоинт и обновили схему, наш сервис автоматически генерирует 10 сущностей для обновленной схемы.

## Шаг 3. Делайте запросы!
Всё готово! Теперь вы можете обращаться к вашему API. Скопируйте ссылку на эндпоинт из интерфейса (она выглядит примерно так: `https://rusapi.fun/api/mock/ВАШ_NANO_ID/users`).

Откройте консоль браузера или ваш код и выполните первый запрос:

```javascript
// 1. Получаем сгенерированные данные
fetch('https://rusapi.fun/api/mock/abc123xyz/users')
  .then(res => res.json())
  .then(data => console.log('Мои пользователи:', data));
