Добро пожаловать в документацию **RusAPI**! 

Если вы фронтенд-разработчик, то наверняка сталкивались с ситуацией: интерфейс уже нужно верстать, а бэкенд еще не готов. **RusAPI** решает эту проблему. Это не просто генератор случайных заглушек, это **полноценная замена бэкенду** на этапе разработки. 

Вам больше не нужно разворачивать локальный `json-server`, писать моки руками или мириться с тем, что данные пропадают после перезагрузки страницы.

## 🔥 Ключевые возможности

* 💾 **Stateful-моки (Живые данные):** Это настоящий бэкенд. Если ваш фронтенд отправляет `POST`-запрос на создание пользователя, этот пользователь реально сохранится в базе данных. При следующем `GET`-запросе вы его увидите. Редактируйте (`PUT/PATCH`) и удаляйте (`DELETE`) данные как в реальном API.
* 🎲 **Умная генерация:** Мы используем всю мощь библиотеки `faker.js`. При создании ресурса достаточно указать типы полей (например, `internet.email` или `image.avatar`), и мы автоматически сгенерируем реалистичные начальные данные.
* 🛡️ **Строгая валидация типов:** RusAPI не позволит превратить моки в свалку. Наш встроенный валидатор проверяет входящие запросы. Если эндпоинт ожидает число, а вы прислали строку — сервер честно вернет ошибку `400 Bad Request`. Лишние поля из запросов автоматически отсекаются.
* 🔑 **Авто-генерация ID:** Вам не нужно думать о первичных ключах. Сервер автоматически присваивает каждому объекту уникальный `id` (формата UUID v4).

## 💻 Как это выглядит на практике?

Никаких сложных настроек аутентификации или токенов. Просто скопируйте URL вашего эндпоинта и делайте обычные запросы.

Пример получения списка данных с использованием современного `async/await`:

```javascript
const fetchUsers = async () => {
  try {
    const response = await fetch('[https://rusapi.fun/api/mock/nano123/users](https://rusapi.fun/api/mock/nano123/users)');
    
    if (!response.ok) {
      throw new Error(`Ошибка HTTP: ${response.status}`);
    }
    
    const users = await response.json();
    console.log('Список пользователей:', users);
  } catch (error) {
    console.error('Ошибка загрузки:', error);
  }
};

fetchUsers();
